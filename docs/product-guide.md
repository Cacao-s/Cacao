# Cacao 產品指南

Cacao 是一套專為「家長 (Giver)」與「孩子 (Baby)」打造的家庭零用金／報銷平台。家長可以設定錢包、定期津貼與審核規則；孩子能在手機上提出請款、追蹤進度並學習理財。此文件描述願景、角色、旅程與 MVP 範圍，作為產品與工程協作的共同語言。

---

## 1. 願景與價值
- **願景**：讓家庭金流透明、好追蹤，並將金錢管理變成家人間的正向對話。
- **核心價值**：
  1. **即時掌握**：家長可一眼知道津貼執行狀況、請款瓶頸。
  2. **養成教育**：孩子在 App 中學習記帳、附上憑證、等待審核。
  3. **安全合規**：所有交易紀錄與通知都有稽核軌跡，方便日後查詢。

---

## 2. 角色與痛點
| 角色 | 需求 | 痛點 |
| --- | --- | --- |
| Giver（父母／監護人） | 設定每週或每月的津貼、快速審核請款、控管預算 | 現在靠聊天軟體與現金，資訊分散、無法追蹤餘額 |
| Baby（子女） | 手機上傳票據、了解審核狀態、收到津貼提醒 | 不知道何時會匯款、附件容易遺失 |
| Admin / Support | 管理帳號、偵測異常、協助客服 | 缺乏統一的審核紀錄與通知系統 |

---

## 3. 主要旅程
### 3.1 Giver 設定津貼
1. 使用電郵登入 Web/行動端。
2. 建立家庭、邀請成員，並創建錢包（貨幣、限額）。
3. 設定 allowance（排程、金額、目標錢包）。
4. 在儀表板查看本週預計支出與孩子請款。

### 3.2 Baby 提出請款
1. 在 App 透過帳號密碼登入。
2. 於「請款」分頁上傳照片、輸入金額與說明。
3. 系統將請款送交 Giver，顯示狀態 `pending`。
4. 審核通過後收到通知並顯示轉帳日期。

### 3.3 Giver 審核請款
1. 打開通知或 Dashboard 進入請款列表。
2. 查看照片、金額與餘額警示。
3. 選擇「核准 / 駁回」，必要時留下理由。
4. 系統寫入交易、更新錢包，並通知 Baby。

## 4. MVP 範圍
| 類別 | 必做 (MVP) | 可延伸 (Post-MVP) |
| --- | --- | --- |
| 身份/登入 | 單一 email + 密碼、單管理者 | 社群登入、2FA、多管理者 |
| 家庭/錢包 | 單一貨幣、多個錢包、錢包餘額檢查 | 多幣別、自動匯率、子錢包共享 |
| 津貼 | 週期性排程、手動觸發、job log | 多條件（表現、勞務）、外部行事曆同步 |
| 請款 | 圖片上傳（暫存於裝置）、金額、備註 | OCR、自動分類、報表匯出 |
| 通知 | App 內提示、Email stub | 推播、LINE Bot/SMS |
| 報表 | Dashboard 總覽、簡單金額指標 | CSV 匯出、稅務報表、AI 洞察 |

## 5. 成功指標 (MVP)
| 指標 | 目標 |
| --- | --- |
| 家庭啟用率 | 首輪封測 20 組家庭完成 onboarding |
| 審核時間 | 平均審核時間 < 2 小時 |
| 請款成功率 | 80% 請款於 48 小時內核准或駁回 |
| App 崩潰率 | < 1% sessions |
| NPS | Beta 使用者 NPS > +30 |

## 6. 體驗設計重點
- **主題與可及性**：預設 Light，支援 Dark 與 High Contrast，必須符合 WCAG AA 對比。
- **語言**：App 與文件預設繁體中文，可切換英文。文案集中於 `apps/mobile/i18n`。
- **通知語調**：友善、鼓勵式，避免指責。例：「太棒了！媽媽已核准你的書籍報銷」。
- **錯誤回饋**：所有錯誤需含原因與下一步（例如請重新上傳清晰照片）。

## 8. 風險與假設
- 家長願意在 App 內處理金流，需提供清楚的資安說明與備援流程。
- 通知若延遲會直接影響體驗，因此需盡早設計非同步重送機制。
- 行動 App 需支援離線草稿（至少保留 3 筆暫存），避免網路不穩導致資料遺失。

## 10. 後續建議
1. 將 Allowance/Request API 製作成 `shared/client-sdk`，供 Web Admin 或外部服務使用。
